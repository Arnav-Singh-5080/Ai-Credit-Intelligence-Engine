##  Getting Started

<details>
<summary><b>Click to expand setup steps</b></summary>

### 1. Fork the repository  
Click on the **Fork** button on GitHub.

### 2. Clone your fork
```bash
git clone https://github.com/your-username/ai-credit-intelligence-engine.git
cd ai-credit-intelligence-engine
```

### 3. Create a new branch
```bash
git checkout -b feature/your-feature-name
```

### 4. Install dependencies
```bash
pip install -r requirements.txt
```

### 5. Run the project
```bash
streamlit run application.py
```

</details>

##  Contribution Workflow

```
Fork → Clone → Create Branch → Make Changes → Commit → Push → Pull Request
```

### Steps:
1. Pick an open issue or create a new one if needed
2. Ask to be assigned  
3. Make your changes  
4. Test your changes  
5. Commit your work  
6. Push to GitHub  
7. Open a Pull Request  

---

##  Coding Guidelines

- Write clean and readable code  
- Follow **PEP8** standards  
- Use meaningful variable names  
- Keep functions small and modular  
- Add comments where needed  
- Do not break existing functionality  

---

##  Pull Request Guidelines

Before submitting a PR:

- Ensure your code works correctly  
- Keep PRs small and focused  
- Write a clear PR title and description  
- Link the issue (example: `Fixes #12`)  
- Make sure your changes are relevant to the issue

### Example PR Title:
```
Add structured and beginner-friendly CONTRIBUTING.md
```

---

##  Beginner Friendly Tips

- Start with issues labeled **good first issue**  
- Don’t hesitate to ask questions  
- Keep changes simple and clear  
- Focus on learning, not perfection
